## 01. 주문 프로세스 (MES 기반)

### 주문 상태 흐름
미입금 → 결제완료 → 제작대기 → 제작중 → 제작완료 → 출고완료 | 주문취소

### 결제 유형
- case1: 카드결제
- case2: 포인트결제
- case3: 에스크로
- case4: 가상계좌 (이니시스 승인)
- case5: 신용결제 (MES에서 생성된 주문, 제작대기에서 시작)

### 파일번호 (고유한 생성번호) 상태

**case1 배송상품**: 상품준비중 → 포장완료
**case2 파일업로드상품**: 확인전 → 파일확인 → [재업로드요청/재업로드완료] → 주문파일 업로드 → 다운로드완료 → 포장완료
**case3 편집기상품**: 렌더링전 → 렌더링완료 → [수정요청/수정완료] → 다운로드완료 → 포장완료

### 출고 유형
- case1 택배출고: 송장출력
- case2 퀵: 납품명세서
- case3 직접방문: 출고명세서

### 고객접점 취소
- 주문취소 가능 시점: 제작대기 이전

- 009: 배송방법이 택배인 주문건이 출고완료로 변경시 - 주문번호/주문자명/송장번호

## 05. Pain Point (현재 시스템 문제점)

### 발주
- OEM업체 주문등록: 비즈하우스 로그인 후 다운 → 해외배송(주소검색) → 색깔종이에 출력
- 후지필름: 생산관리 프로그램에서 생산대기 선택 후 다운 → EXCEL로 변환 후 등록
- 켄터1hr: 팬허드에서 EXCEL 확인(수동주문이라고 생각여길) → 직접 다감
- 자사사이트 주문확인

### 파일확인/파일명
- 비즈하우스: 자유형반칼스티커(AI) - 조판원료된 형태
- 후지필름: 규격반칼스티커(JPG), 아크릴상품(JPG)
- 디지털인쇄상품: 종이명/사양을 파일명에 추가 (*캘린더, 떡메모지)
- 자사사이트: 대부분 시 또는 JPEG 업로드됨, 다도안파일이 많음
- 출력파일(PDF) 생성시 페이지 배열, 씩메모지 등 판걸이 명시

### 출력/조판/외주발주
- 사용자 Pain: 장바구니가 없음→의뢰서에 메모해서 수동으로 처리, 주문을 건별로 입금은 합산하여 입금→무통장 입금 확인대기가 하루 다감, 주문완료와 파일을 별하드에 올림→핸드사용이 어려움, 제주택배비 및 추가배송비 부과→전화로 연락하여 받고 있음
- 출력: 판매처별로 상품/소재/출력사이즈가 다르게 표기, 다도안으로 인한 판걸이(인포지선)이 달라 구분이 어려움, 출력시간 분배가 어려움
- 출고: 합배송의 리스트를 알수없음, 송장주가시 배송정보를 입력해야함, 하나의 주문에 여러개의 송장부여 가능하도록

## 07. 클레임처리
- 재제작+재배송: case1 일부재제작+재배송, case2 전체재제작+재배송
- 주문취소(환불): case1 부분환불→취소처리안함(매출과 맞지않음)(이니시스에서 부분취소 또는 무통장), case2 전체환불→주문취소처리(이니시스에서 취소 또는 무통장)
- 반송: Nfocus에서 반송요청
- 재출고: 신규송장부여, case1 상품누락으로 추가발송, case2 교환으로 인한 재발송
- 주문생성으로 해결: MES내에서 주문복사 또는 오프라인주문으로 처리, front페이지와 연동되지 않아도 됨, 반송여부만 체크
- 정산관련: 주문내에 정산관련 메모필드 추가, 반송비 부담, 재제작 비용부담

주문정보에 세금계산서 발행여부와 발행날짜 정보 → 추후 고도화

## 09. 발주업무 프로세스
- 주문상태감: 제작대기 → 제작중
- 발주 업무프로세스: 접수파일등록 → server(썸네일생성) → 의뢰서출력 → 주문상태변경 → 외주발주
- 외주발주: 01. 접수파일을 모아 외주처별 발주서 생성 ► 발주서, 02. 책자상품의 경우 수량, 크기에 따라 선택적 외주발주
- 수동조판: 여러주문을 모아 수동조판 (ex.틴거울, 아크릴상품, 자석북마크) ► 수동조판한 파일은 별도전달

## 10. 주문상태별 업무 프로세스

### CS(접수팀) 워크플로우
1. 주문등록: 주문관리>[업체별 주문등록], 엑셀등록/파일경로선택, 데이터 검증
2. 파일처리: 주문관리>[주문정보관리]>주문별>(우측)주문상세, 원본파일이 시면 PDF로 변환 업로드, 인쇄파일과 칼선파일을 분리하여 공정별 등록, 파일명 체크, 출고예정일 등록, 생산메모, 주문번호별 Sort
3. 의뢰서 출력 → 제작 대기
4. 파일다운로드: 접수파일 파일명 그대로 받아짐
5. 제작완료바코드입력
6. 송장출력

### 서버(자동화) 상태 흐름
주문완료 → 접수완료 → 제작대기 → 제작중 → 출고준비중 → 출고완료
- 접수완료: 업로드 시 접수파일 중 인쇄공정의 파일로 자동 썸네일 생성
- 제작대기: 의뢰서출력하면 빨간점 표시
- 제작중: 제작중으로 상태변경팀
- 출고준비중: 송장부여
